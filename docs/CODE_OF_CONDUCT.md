# Code of Conduct

This project follows a short and direct code of conduct.

## What we expect

- Be respectful in issues, PRs, and reviews. Disagreement is fine, contempt is not.
- Keep discussions on-topic. smolbench cares about LLM benchmarks for app workloads. Politics, religion, and personal disputes belong elsewhere.
- Assume good faith on first interaction. Most rough comments are tone, not malice.
- Credit prior work when you build on it.

## What is not tolerated

- Harassment, slurs, threats, doxxing, or sexualised content directed at any contributor.
- Knowingly malicious PRs (backdoors, supply-chain attacks, credential exfiltration).
- Repeated bad-faith arguments after a maintainer has explained the project's position.

## Enforcement

The maintainer (`@ShageeshanT`) reviews reports and may:

1. Edit or remove offending comments.
2. Lock or close the issue or PR.
3. Block the user from the repository.

Action is at the maintainer's discretion. Repeat offenses lead to permanent bans.

## Reporting

Open a GitHub issue tagged `coc`, or contact the maintainer privately (security advisory works for sensitive reports). Reporters' identities will not be shared without consent.
